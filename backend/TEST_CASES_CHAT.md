# 智能问答模块测试用例清单

> 配套自动化脚本：`python run_chat_tests.py`  
> 用例数据：`test_cases_chat.json`  
> 建议测试账号：`emp001` / `123456`（普通员工）

## 一、测试前准备

| 步骤 | 命令 / 操作 |
|------|-------------|
| 1. 初始化数据库 | `cd backend && python init_db.py` |
| 2. 启动后端 | `python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload` |
| 3. 启动前端 | `cd frontend && npm run dev` |
| 4. 登录 | http://localhost:3000 → `emp001` / `123456` → 智能问答 |

**判定标准：** 每条回复旁的标签（`answer_type`）与下表一致，且回答包含「期望要点」中的关键信息。

## 六、工单与衔接 — 10 个场景

每个场景在**同一对话**中按顺序输入。标签对照：工单申请=`ticket_clarification`，待确认=`ticket_confirm`，已提交=`ticket_submitted`，工单咨询=`ticket_qa`。

### T-01 在职证明完整提交

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 我要开一份在职证明 | 工单申请，追问用途/接收单位等 |
| 2 | 用来办签证，交给日本领事馆，要盖章，最好这周五前 | 待确认工单，含 **日本**、**签证** |
| 3 | 点击「确认提交」或输入「确认提交」 | 工单已提交，含 **TK** 编号 |

### T-02 考勤异常提交

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 我要提交考勤异常说明 | 追问异常日期/类型/原因 |
| 2 | 2026年7月3日, 打卡缺失, 忘记打卡 | 待确认，3 个字段齐全 |
| 3 | 确认提交 | 工单已提交 |

### T-03 确认前修改字段

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1–2 | 同 T-01 前两步 | — |
| 3 | 接收单位改成美国大使馆 | 接收单位更新为 **美国** |

### T-04 未指明工单类型

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 提交一个新工单 | 列出工单类型供选择 |
| 2 | 1（或「在职证明」） | 进入证明开具流程 |

### T-05 工单 ↔ 普通问答切换

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 我要开一份在职证明 | 进入工单 |
| 2 | HR电话是多少？ | **规则答案**，含 88888888 |
| 3 | 继续修改 | 回到工单草稿/修改模式 |

### T-06 提交后追问

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1–3 | 同 T-01 完整提交 | — |
| 4 | 这个大概要等几天？ | 工单咨询，含 **工作日** |

### T-07 信息变更识别

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 我想修改手机号 | 进入信息变更，追问变更项/原值/新值 |

### T-08 提交前校验

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1–2 | 同 T-01 前两步 | — |
| 3 | 这样可以吗？ | 提示信息 **完整**，引导 **确认** |

### T-09 考勤异常修改原因

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1–2 | 同 T-02 前两步 | — |
| 3 | 异常原因改成: 路上堵车 | 原因更新为 **堵车** |

### T-10 规则回答后追问

| 步骤 | 输入 | 期望 |
|------|------|------|
| 1 | 忘记打卡怎么办？ | 规则/标准答案，含 **补卡** |
| 2 | 那迟到呢？ | 回答 **迟到** 处理（绩效/旷工） |

## 七、手工验证工单落库

完成 T-01 或 T-02 后：

1. 进入 **我的 → 人工请求**
2. 列表应出现新工单，类型为中文（如「证明开具」「考勤异常」）
3. 点击行或「详情」可查看完整说明

## 八、自动化测试命令

```powershell
cd backend

# 仅单元测试（无需启动服务）
python run_chat_tests.py --unit-only

# 仅 API 集成测试（需后端已启动）
python run_chat_tests.py --api-only

# 全部测试 + 生成报告
python run_chat_tests.py --report test_chat_report.json
```

报告默认输出：`backend/test_chat_report.json`
