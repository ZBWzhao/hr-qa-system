## 四、测试数据说明

### 文档隔离机制
1. **数据库层面**：`hr_document` 表的 `department_id` 字段标记文档归属部门
2. **查询层面**：`documents.py` 中的 `list_documents` 函数根据用户 `department_id` 过滤文档
3. **向量检索层面**：`vectorstore.py` 中的 `search_similar` 函数使用 `where_filter` 过滤部门

### 向量数据库过滤逻辑
```python
if department_id:
    # 0 表示全公司通用制度；部门用户也应能检索到
    where_filter = {"department_id": {"$in": [int(department_id), 0]}}
```

- `department_id = 0`：全公司通用文档，所有部门可见
- `department_id = N`：部门专属文档，仅该部门可见

---

## 五、测试注意事项

1. **通用文档**：如果存在 `department_id = None` 或 `department_id = 0` 的文档，所有部门都能看到
2. **向量库同步**：确保文档已发布（status = "published"）并已建立向量索引
3. **RAG检索**：AI回答依赖RAG检索，如果向量库为空可能无法回答
4. **模型能力**：大模型可能会"编造"信息，需要验证回答是否真的来自文档
